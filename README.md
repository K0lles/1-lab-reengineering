1. **StringBuilder замість конкатенації рядків:** Конкатенація рядків через + всередині циклів є неефективною, оскільки кожен раз створюється новий об'єкт String. Використання StringBuilder дозволяє зменшити накладні витрати на виділення пам'яті та збільшити швидкодію при роботі з великими текстовими даними.<br/><br/>
2. **HashMap для підрахунку частот:** Замість пошуку та порівняння слів через distinctString.contains(), ми використовуємо HashMap, де ключем є слово, а значенням — кількість його появ. Це значно швидше, оскільки пошук та вставка в HashMap працюють за O(1) в середньому випадку, що суттєво підвищує ефективність обчислень.<br/><br/>
3. **Stream API для сортування:** Використання Stream API дозволяє спростити та прискорити сортування слів за їх частотністю, оскільки сортування по мапі працює ефективніше, ніж пряме порівняння рядків.<br/><br/>
4. **Math.min для обмеження кількості результатів:** Замінюємо жорстке обмеження в циклі на Math.min(30, sortedEntries.size()), що дозволяє коректно працювати навіть якщо у файлі менше ніж 30 унікальних слів.
